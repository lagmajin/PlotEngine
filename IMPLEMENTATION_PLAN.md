# PlotEngine 実装計画書

> 最終更新: 2026-05-16
> 目標: Venice を主系にした、AI 直接編集対応の小説制作 IDE に仕上げる

---

## 現状の前提

この計画は、いまの実装状態を前提にしています。

- `vcpkg` manifest mode は導入済み
- C++20/23 modules はプロジェクト方針として必須
- `import std;` はプロジェクト所有の module unit で必須
- Qt UI は Verdigris (`W_OBJECT`) を使って `moc` を使わない方針
- `qt-advanced-docking-system` は IDE シェルの基盤として導入済み
- Venice は AI の主系
- API key は DPAPI で安全に保存する
- `NovelEditor` は IDE 風のテキストエディタとして稼働中
- `MainWindow` は module 化済みで、レイアウト保存、最近のプロジェクト、ウェルカム画面を持つ

## 全体戦略

1. AI に「文章を生成させる」より、「構造化された編集命令を返させる」
2. UI は見た目より導線を優先する
3. すべての変更は diff と rollback を持つ
4. モジュール方針は崩さず、モジュール境界を細くして安定化する
5. Venice を主系に固定し、他プロバイダは互換・予備に留める

## 技術方針

### モジュール境界
- `PlotEngine.Core.*` - ドメインロジック、シリアライズ、プロジェクト操作
- `PlotEngine.AI.*` - AI provider、prompt builder、direct edit action
- `PlotEngine.Export.*` - 出力ロジック
- UI は当面 `MainWindow` / `Panel` / `Editor` の単位で維持する

### AI 方針
- Venice を主系にする
- AI は JSON action を返す
- 直接編集は必ずアプリ側が検証して適用する
- 危険な変更はプレビュー必須

### 安全方針
- API key は平文保存しない
- データ変更にはログを残す
- rollback を前提にする
