# Agents.md

PlotEngine の開発方針です。既存動作を壊さず、小さく前進させることを優先します。

## 開発方針

- 変更は小さく、局所的に入れる
- 既存の挙動を維持する
- 大きな設計変更は、段階を分けて進める

## モジュール方針

- `ixx` を基本方針とする
- 新規のドメインコード、サービスコード、切り出し可能な共通処理は、まず `ixx` 化を検討する
- 既存コードの移行は、影響範囲の小さい単位から進める
- Qt 依存の強い UI ハブや `MainWindow` のような集約点は、必要なら当面非モジュールのままでもよい
- ただし、UI でも独立しやすい部品は優先的にモジュール化する
- いったん `ixx` 化した領域を classic header/source に戻すのは禁止
- 例外は、ユーザーが明示的に classic へ戻すよう指示した場合だけとする

## `import std`

- プロジェクト所有のモジュール単位では `import std;` を基本とする
- 可能なら `#include` より `import` を優先する
- 既存の非モジュールコードは、無理に一度で置き換えない

## 進め方

- 迷ったら、より小さい変更で済む方を選ぶ
- 実装変更とアーキテクチャ変更を同じ差分に詰め込みすぎない
- モジュール境界は、ビルドと責務が安定する範囲から広げる
